# Metadata Policy

## Core Principle

Every asset in the system must carry structured metadata. Assets without valid metadata cannot be registered, retrieved, or promoted.

## Required Fields

These fields are mandatory for **every** asset (as defined in `schemas/asset_card.schema.json`):

| Field | Type | Description |
|-------|------|-------------|
| `asset_id` | string | Globally unique identifier (generated by `tools/utilities/id_generator.py`) |
| `title` | string | Human-readable title |
| `asset_type` | enum | One of: `code`, `logic`, `template`, `doc`, `failed_case` |
| `language` | enum | One of: `sas`, `r`, `sql`, `python`, `markdown`, `yaml`, `json` |
| `status` | enum | One of: `draft`, `tested`, `reviewer_approved`, `production_approved`, `deprecated` |
| `tags` | array[string] | Searchable tags (e.g., `["ae_summary", "teae", "big_n"]`) |
| `created_by` | string | Author identifier |
| `created_at` | string | ISO 8601 timestamp |
| `version` | string | Semantic version (e.g., `1.0.0`) |

## Optional but Recommended Fields

| Field | Type | Description |
|-------|------|-------------|
| `description` | string | What this asset does |
| `task_types` | array[string] | Which task types this asset supports |
| `dependencies` | array[string] | Asset IDs this asset depends on |
| `source_doc` | string | Reference document (SAP, spec, etc.) |
| `updated_at` | string | Last modification timestamp |
| `updated_by` | string | Who last modified |
| `review_notes` | array[string] | IDs of associated review notes |
| `promotion_history` | array[object] | History of status changes |

## Validation

- Metadata is validated by `tools/validators/metadata_validator.py` against `schemas/asset_card.schema.json`.
- Required fields are configured in `config/governance/metadata_required_fields.yaml`.
- Validation happens at registration time and before every promotion.
