# Non-Goals

These are things we will **not** pursue, even in later phases, to keep the project focused:

- **Replacing human programmers**: This system augments, not replaces, clinical programmers.
- **Regulatory submission tool**: This is a knowledge base, not a submission-ready system.
- **General-purpose code generation**: Only clinical programming use cases are targeted.
- **Universal LLM agent framework**: We build only the workflows needed for clinical coding tasks.
- **Competing with commercial CDISC tools**: Focus is on internal knowledge reuse, not standards compliance tooling.
