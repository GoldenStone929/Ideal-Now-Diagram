# Asset Metadata Standard

## Purpose

Defines the metadata structure that every asset must carry. This standard is enforced by `schemas/asset_card.schema.json` and validated by `tools/validators/metadata_validator.py`.

## Field Definitions

### Identity Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `asset_id` | string (UUID) | Yes | Globally unique; generated by `id_generator.py` |
| `title` | string | Yes | Concise, descriptive title |
| `description` | string | Recommended | What the asset does or represents |
| `version` | string (semver) | Yes | Semantic version, starting at `1.0.0` |

### Classification Fields

| Field | Type | Required | Values |
|-------|------|----------|--------|
| `asset_type` | enum | Yes | `code`, `logic`, `template`, `doc`, `failed_case` |
| `language` | enum | Yes | `sas`, `r`, `sql`, `python`, `markdown`, `yaml`, `json` |
| `tags` | array[string] | Yes (≥1) | Free-text tags for search |
| `task_types` | array[enum] | Recommended | `ae_summary`, `lab_shift`, `disposition`, `listing`, `qc_compare` |

### Lifecycle Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `status` | enum | Yes | `draft`, `tested`, `reviewer_approved`, `production_approved`, `deprecated` |
| `created_by` | string | Yes | Author identifier |
| `created_at` | string (ISO 8601) | Yes | Creation timestamp |
| `updated_by` | string | On update | Who last modified |
| `updated_at` | string (ISO 8601) | On update | Last modification timestamp |

### Relationship Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `dependencies` | array[string] | No | Asset IDs this asset depends on |
| `source_doc` | string | Recommended | Reference document identifier |
| `review_notes` | array[string] | No | IDs of associated review notes |
| `promotion_history` | array[object] | Auto-filled | History of status changes |

## Versioning Rules

- Metadata-only changes: patch version bump (e.g., `1.0.0` → `1.0.1`).
- Logic or content changes: minor version bump (e.g., `1.0.0` → `1.1.0`).
- Breaking changes (interface, dependencies): major version bump (e.g., `1.0.0` → `2.0.0`).
